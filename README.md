# Surface, Explore, Record

A methodology for working with AI coding assistants that helps you uncover hidden decisions, leading to more reliable, production-ready applications.

## About This Project

When you ask AI to build an app, it makes dozens of silent decisions about architecture, data storage, authentication, error handling, and more. These decisions happen invisibly, and you only discover problems when something breaks in production.

**Surface, Explore, Record** changes that dynamic by treating AI as an analyst on your team rather than a vending machine that outputs code.

### The Three Steps

1. **Surface** the decisions: Ask AI to identify what decisions need to be made at your current stage
2. **Explore** the options: For each decision, understand the strengths, weaknesses, and tradeoffs
3. **Record** the choices: Document what you decided and why, creating a foundation for everything that follows

## The Application

Throughout this series, I build a task management app called **TaskFlow** using this methodology. Each episode covers a different stage of the software development lifecycle, demonstrating how to stay in control of AI-assisted development.

## Key Concepts

### Why This Matters

Silent decisions create the gap between "AI built me something fast" and "AI built me something that works for me." By surfacing these decisions, exploring your options, and recording your choices, you:

- Understand what you are building and why
- Avoid expensive rework from hidden architectural choices
- Create documentation that makes future AI interactions more effective
- Stay in control throughout the entire development lifecycle

### When to Use This Framework

- **Starting a new project**: Surface decisions before any code is written
- **Mid-project**: Pause and audit what decisions have been made
- **Before deployment**: Understand what is in your app before real users arrive
- **Adding features**: Check how new work interacts with existing decisions
