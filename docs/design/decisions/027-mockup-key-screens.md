# Decision 027: Key Screens for Mockups

**Status:** Decided

**Category:** Mockup Scope

**Question:** Which screens should be included in the mockup website? Candidates include: login/sign-up, dashboard, board view, board view with task detail panel open, list view, settings, notifications, and empty states. Which are essential for evaluating each design direction?

**Decision:** Four key screens for the mockup website:

1. **Login / Sign Up** — Establishes the brand's first impression. Shows the logo, typography, form inputs, and primary button styling.
2. **Dashboard** — Shows the full application shell (sidebar, header, content area) with the three dashboard sections (My Tasks, Recent Activity, Projects). Demonstrates layout, avatars, status badges, and empty vs populated states.
3. **Board View** — The core screen. Shows the sidebar, header with breadcrumbs, project sub-nav (filters, sort, view toggle), and Kanban columns with task cards. Demonstrates the column layout, card design, status colors, priority icons, label chips, and avatars.
4. **Board View with Task Detail Panel** — The board view with the side panel open, showing the task detail layout: title, properties, description (with Markdown rendering), and comments thread. Demonstrates the panel overlay, form controls, and comment formatting.

Each screen is shown in all three design theme variants for comparison.

**Rationale:** These four screens cover the full range of TaskFlow's visual design surface: authentication (login), navigation and layout (dashboard), the primary workspace (board), and the detail interaction (task panel). Together they exercise every major component: buttons, inputs, cards, badges, avatars, dropdowns, panels, and typography. The login screen is included because first impressions matter for a demonstration project. The board with panel open is separated from the plain board because it's visually dense and exercises different components.
