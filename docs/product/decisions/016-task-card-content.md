# Decision 016: Task Card Content

**Status:** Decided

**Category:** Board & Views

**Question:** What information is visible on a task card on the board without opening it — just the title, or also assignee avatar, labels, due date, comment count, etc.?

**Decision:** Task cards display: title, assignee avatar, label chips (colored), due date (if set, with overdue visual indicator), priority indicator, and comment count. The task description is not shown on the card — users open the task detail view for that.

**Rationale:** These fields give users enough at-a-glance information to understand what a task is, who owns it, how urgent it is, and whether there's discussion happening — without opening every card. Excluding the description keeps cards compact so more tasks fit on the board. This set aligns with the task fields established in the BRD (title, assignee, labels, due date, priority) and the decisions made in 005 (due dates), 006 (priority), and 014 (label colors).
